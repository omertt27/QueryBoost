Privacy Policy

QueryBoost Privacy Policy

Plain language. No legalese. No surprises.

Last updated: March 25, 2026 · Applies to QueryBoost v2.1.0 and later

This policy is also published publicly at: github.com/omerTT/queryboost/blob/main/privacy.html

Plain-English Summary

  • ✓ QueryBoost runs entirely inside your browser. No data ever leaves your device.
  • ✓ Your queries are read locally only to detect intent and inject an enhancement wrapper.
  • ✓ Settings and feedback are stored locally using Chrome's built-in storage APIs.
  • ✗ QueryBoost does not have a backend, server, or API endpoint.
  • ✗ QueryBoost does not transmit, log, or share your queries or any personal data.
  • ✗ QueryBoost does not use analytics, telemetry, crash reporting, or tracking pixels.
  • ✗ QueryBoost does not use cookies or any cross-site tracking mechanism.

1. What QueryBoost Accesses

When you submit a query on a supported AI platform (ChatGPT, Claude, Gemini, or Perplexity), QueryBoost reads the text of that query in order to:

  • Classify the intent type (e.g. code, how-to, analysis)
  • Generate an enhanced prompt by appending structured instructions
  • Re-inject the enhanced query into the input field before submission

This processing happens entirely within the content script running in your browser tab. The query text is never sent anywhere outside of the page you are already on.

2. Data Stored on Your Device

QueryBoost stores a small amount of data locally using Chrome's storage.sync and storage.local APIs. This data never leaves your browser except via Chrome's own account sync (which you control in your Chrome settings).

Key What it stores Where
qb_enabled Whether the extension is on or off sync
qb_domain_mode Your selected persona mode (general, developer, etc.) sync
qb_ab_variant Your randomly-assigned A/B wrapper variant (A or B) sync
qb_transparency Whether wrapper transparency mode is enabled sync
qb_custom_wrappers Your custom wrapper templates (text you wrote) sync
qb_last_type The type label from your last boost (e.g. "Code / Debug") sync
qb_boost_count Lifetime count of queries boosted sync
qb_feedback Your 👍/👎 ratings — up to 500 entries (type, variant, platform, timestamp) local
qb_ab_stats Aggregate sent/thumbs-up/thumbs-down counts per variant local

Note: Feedback entries include the query type (e.g. "code"), the platform (e.g. "chatgpt"), and a timestamp — but never the query text itself.

3. Chrome Permissions Used

  • activeTab — Allows the popup to detect which supported platform you're on.
  • storage — Allows saving your settings and feedback locally (described above).
  • host_permissions (claude.ai, chatgpt.com, gemini.google.com, perplexity.ai) — Required to run the content script on these pages to intercept and enhance queries.

No other permissions are requested. QueryBoost does not request access to browsing history, bookmarks, cookies, downloads, or any other browser data.

4. No External Communication

QueryBoost has no backend. It does not make any network requests to any server owned or operated by the developers. The extension is entirely self-contained: a collection of local JavaScript and HTML files that run inside your browser.

The only network activity associated with QueryBoost is the query you submit to ChatGPT, Claude, Gemini, or Perplexity — which goes directly to those services, exactly as it would without the extension, just with the enhanced text.

5. Chrome Account Sync

Settings stored in chrome.storage.sync (your toggle state, domain mode, A/B variant, custom wrappers) may be synced across your Chrome-signed-in devices if you have Chrome Sync enabled. This sync is handled entirely by Google Chrome — QueryBoost has no involvement in or visibility into this process.

To prevent syncing, you can disable extension data sync in Chrome settings, or use the extension in a profile without Chrome Sync enabled.

6. Deleting Your Data

You can delete all data stored by QueryBoost at any time:

  • Feedback and A/B stats — Open the popup → Settings tab → click "Clear feedback & A/B data".
  • Custom wrappers — Open the popup → Custom tab → select each type and click "Reset to default".
  • All data — Uninstalling the extension removes all locally stored data. Synced data can be cleared in Chrome's extension storage settings.

7. Children's Privacy

QueryBoost does not knowingly collect any information from anyone, including children under the age of 13. As the extension collects no personal data at all, it has no specific provisions for children's data.

8. Changes to This Policy

If this policy changes in a meaningful way (e.g. if a future version adds any form of data collection), the changelog will describe it clearly and the "Last updated" date above will be updated. We will not retroactively apply a new policy to data collected under a prior policy.
